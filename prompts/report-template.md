# Report Template — 최종 리포트 마크다운 템플릿

> **용도**: `analyze-ingredient.md`가 생성한 JSON 분석 결과를 **사용자 가독성 마크다운 리포트**로 변환.
> **호출 시점**: SKILL.md Phase 5 (리포트 생성).
> **입력**: `analyze-ingredient.md` 출력 JSON + 사용자가 선택한 출력 언어(`kor` / `en` / `both`).
> **출력**: 마크다운 리포트 (사용자에게 그대로 노출).
> **참조**: `references/disclaimer.md` (면책 + 경고 배너 자동 삽입).

## 3. 섹션별 렌더링 규칙

### 3-1. `{{WARNING_BANNERS}}` — 조건부 배너 (최상단 필수)

`analyze-ingredient.md`의 `warnings_triggered` 필드 기반. 우선순위 순으로 삽입 (`references/disclaimer.md` §3):

1. `infant_or_pet: true` → §3-2 배너만 출력 + **이후 전체 리포트 생략 (거부)**
2. `pregnancy: true` OR `breastfeeding: true` → §3-1 배너
3. `prescription_drug_present: true` → §3-3 배너
4. `ul_exceeded.length > 0` → §3-4 배너 (성분명 `[성분]` 치환)

해당 없으면 빈 문자열로 치환.

### 3-2. `{{EXEC_SUMMARY}}` — 분석 요약 결론

**형식**: 서술형 단락 (불릿 X), **약 500자 이내 (300~550자 권장)**. 전체 맥락을 아우르는 결론으로 사용자가 이 한 단락만 읽어도 "무엇을 왜 해야 하는지" 이해 가능해야 함.

**필수 포함 요소** (순서대로):
1. **현 스택 전반 평가** — 몇 개 제품, 어떤 카테고리, 전반적 균형 상태
2. **가장 중요한 문제 1~2개** — 상한섭취량 초과·중복·결핍 중 긴급한 것, **현재 복용 중인 제품명**과 **한국어 영양소명**으로 특정
3. **사용자 프로필·라이프스타일 반영** — (해당 시) 임신/음주/복용약/증상이 결론에 어떻게 연결되는지
4. **행동 권고 2~3개** — 각 권고는 **현재 복용 중인 제품명 중심**. "마그네슘을 줄이세요"(X) → "CoreBone Calci-Mag-D의 마그네슘 형태(산화마그네슘)를 글리시네이트로 교체하세요"(O)

**톤**: 친구가 설명해주듯 자연스럽게. 전문용어는 한 번 쓰고 괄호로 풀어쓰기.

**예시 (40대 초 남성, 잦은 음주, 6제품 스택)**:
```
현재 간건강·뼈건강·비타민D·오메가3·종합비타민·유산균 6종을 드시고 계시는데, 전반적으로는 커버 범위가 넓지만 두 가지 과잉과 한 가지 형태 문제가 있습니다. 우선 **SunDaily D3 5000IU**는 단독으로도 상한섭취량(UL)의 125%인데, CoreBone Calci-Mag-D · DailyCore Multi에도 비타민D가 들어 있어 총 160%까지 초과됩니다 — 이 제품을 중단하시거나 격일 복용으로 줄이시는 것이 최우선입니다. 두 번째로 **CoreBone Calci-Mag-D의 마그네슘**은 산화마그네슘 형태로 흡수율이 약 4%에 불과해, 총량은 상한을 넘는데 실제 몸에 들어오는 양은 적습니다 — 글리시네이트 형태의 마그네슘 보충제로 교체하시면 양과 형태 문제가 동시에 해결됩니다. 음주가 잦으신 점을 고려하면 **GentleLiver Relax(실리마린)**는 유지하시거나 흡수율이 더 좋은 파이토솜 형태로 업그레이드하시는 것이 좋고, 칼슘·아연은 식단(유제품·굴·소고기)으로 보완하세요.
```

**필수 규칙** (§8 금지사항 연동):
- 약 500자(±100자). 너무 짧으면 맥락 부족, 너무 길면 요약 기능 상실
- **원소 기호 단독 사용 금지** — Mg(X), Ca(X), Zn(X) → 마그네슘·칼슘·아연 (§8 금지사항 #13)
- **액션은 제품명 중심** — 사용자가 실물을 특정할 수 있어야 함 (§8 금지사항 #14)
- 제품명은 **사용자가 준 표현 그대로** 사용 (예: "SunDaily D3 5000IU", "CoreBone Calci-Mag-D")
- 첫 줄은 "현재 ~ 드시고 계시는데" 같은 **현황 요약**으로 시작, 사용자 맥락에서 출발

**권장 가이드라인** (§8-B 작성 가이드 참조, 상황에 맞게 적용):
- 상충되는 지표(예: 기준 초과이면서 실제 흡수는 부족)가 있을 때는 결론 문장을 먼저 쓰면 사용자가 이해하기 쉬움. 단, 단일 방향 지표(명확한 초과 or 명확한 부족)에는 굳이 강조 불필요
- 선택이 필요한 상황(예산·효능 트레이드오프)은 EXEC_SUMMARY에서 "선택이 필요합니다" 정도로만 언급하고 A/B 비교는 `{{STACK_OPTIMIZATION_SECTION}}`에서. 단, 명확한 정답이 있는 교체(예: 산화마그네슘 → 글리시네이트)는 단독 권고가 적절

```markdown

|------|--------|------|------|--------|-----------|
| 비타민 D | 150 mcg | 1000% | 150% | 🔴 | 2개 제품 중복 + UL 초과. 장기 복용 시 고칼슘혈증 위험 [출처: Tier 1 / https://ods.od.nih.gov/factsheets/VitaminD-HealthProfessional/] |
| 비타민 B6 | 50 mg | 3846% | 50% | 🟠 | UL의 50%. 장기 고용량 시 감각신경병증 보고 [출처: Tier 1 / https://ods.od.nih.gov/factsheets/VitaminB6-HealthProfessional/] |
| EPA+DHA | 700 mg | — | — | ℹ️ | 심혈관 건강 일반 섭취 권장 범위 250-1000mg [출처: Tier 1 / https://ods.od.nih.gov/factsheets/Omega3FattyAcids-HealthProfessional/] |
```

**규칙**:
- RDA/UL 미설정 성분은 `—` + `ℹ️` 플래그 + `reference_dose_range` 인용
- 코멘트 뒤 `[출처: Tier N / URL]` 필수. 근거 없으면 `"근거 부족 (검색 0건)"`
- 출처 URL이 너무 길면 **전문 URL 유지** (단축 X)

### 3-5. `{{CONTRIBUTION_TABLE}}` — 제품별 기여도 (복수 제품 전용)

중복/공유 성분만 표시. 단일 제품에만 있는 성분은 스택 요약표로 충분.

```markdown
| 성분 | 합산 | 제품 1 (DailyCore) | 제품 2 (PureSea) | 제품 3 (SunD3) |
|------|------|-------------------|-----------------|----------------|
| 비타민 D | 150 mcg | 25 mcg | — | 125 mcg |
| 칼슘 | 200 mg | 200 mg | — | — |
```

- `—`는 해당 제품에 미포함
- **2개 이상 제품에 등장하는 성분만** 이 표에 표시

### 3-6. `{{DUPLICATES_SECTION}}` — 중복 검출 (복수 제품 전용)

```markdown
### 🔴 중복 섭취 경고

- **비타민 D**: 2개 제품(DailyCore + SunD3)에 중복 함유, 합산 150 mcg (UL 100 mcg의 **1.5배 초과**)
 - 권고: SunD3 중단 또는 격일 복용 전환
```

### 3-6a. `{{UL_EXCEEDED_SECTION}}` — UL 초과·근접 상세

`stack_analysis[]`에서 `flag ∈ {"🔴", "🟠"}` 인 항목을 심각도 순으로 렌더.
**각 항목에 `health_impact_ref`의 `summary_short` + `threshold_note` 인용** → 사용자가 "왜 위험한지" 구체적으로 이해.

```markdown
### 🔴 비타민 D — UL 150% 초과
- **합산 섭취**: 150 mcg / 일 (UL 100 mcg)
- **제품 기여**: DailyCore 25 mcg + SunD3 125 mcg
- **건강 영향 (초과 시)**:
 - 단기 고용량 시 오심·식욕부진·다뇨
 - 장기 UL 초과 시 **고칼슘혈증 → 신결석·신석회화**, 혈관 석회화
 - 위험 임계값: 혈중 25(OH)D > 100 ng/mL에서 독성
 - [출처: Tier 1 / https://ods.od.nih.gov/factsheets/VitaminD-HealthProfessional/]
 - 상세 카탈로그: `references/deficiency-excess-effects.md §2.2`
- **조치**: SunD3 중단 또는 격일 복용

### 🟠 마그네슘 (보충제 기준) — UL 113%
- **합산 섭취**: 400 mg / 일 (보충제 UL 350 mg, 식단 제외)
- **건강 영향 (보충제 UL 초과 시)**:
 - 설사·복부경련·오심 (보충제 기준 UL의 목적 자체)
 - 신장 기능 저하자에서 고마그네슘혈증 위험
 - [출처: Tier 1 / https://ods.od.nih.gov/factsheets/Magnesium-HealthProfessional/]
 - 상세 카탈로그: `references/deficiency-excess-effects.md §17.2`
- **조치**: 제품 용량 절반으로 감량 또는 격일 복용
```

**규칙**:
- 🔴 → 🟠 순으로 정렬
- 건강 영향은 **단기 / 장기 / 위험 임계값 / 고위험군** 중 해당하는 것 최소 2개 불릿
- 카탈로그 section (`§N.M`) 반드시 표기 → 사용자가 원본 열람 가능
- `health_impact_ref == null`인 경우 (효과 카탈로그 미등재): "효과 카탈로그 미등재 — 일반적으로 수용성 영양소이나 장기 고용량은 권장 안 함" 형식의 보수적 문구

### 3-6b. `{{DEFICIENCY_SECTION}}` — RDA 미달 상세

`deficiency_gap[]` 항목을 심각도 순으로 렌더. **보충제 스택 기준 부족**임을 명시하고, 식단 보완 가능성을 함께 안내.


**필수 규칙**:
- 3종 중 비어있는 그룹은 "해당 없음"이 아닌 **서브헤더 전체 생략**
- 단일 제품일 때 `stack_optimization`이 대체로 빈 배열 → 단일 제품 기반 remove/reduce만 출력

 - ❌ "마그네슘 300mg — 흡수율 낮음, 형태 교체 필요"
 - ✅ "**CoreBone Calci-Mag-D** — 마그네슘 형태가 산화마그네슘(흡수율 ~4%)이라 글리시네이트 형태의 단독 제품으로 교체 권장"
- **원소 기호 금지** (§8 금지 #13) — 한국어 정식 명칭 사용

**권장 가이드라인** (§8-B, 상황 판단):
- **단독 권고 vs A/B 비교**는 스킬이 판단:
 - **단독 권고**가 적절한 경우 — 명확한 정답이 있는 교체 (예: 산화마그네슘 → 글리시네이트 형태, 비타민 D 중복 제거), 안전성 이슈, 비용·접근성 차이가 미미한 경우
 - **A/B 비교표 + 사용자 선택 질문**이 적절한 경우 — 예산·효능·접근성이 뚜렷하게 충돌하는 경우 (예: 현 제품 재구매 2만원 vs 파이토솜 업그레이드 6만원, 흡수율 7~10배 차이), 사용자 선호가 결정 요인인 경우
 - 비교표 제시 시 컬럼: 선택지명 / 제품 예시 / 장점 / 단점 / 월 비용 대략 + 마무리 1문장 "어느 쪽으로 진행할지 알려주세요"
- 애매하면 **단독 권고** 기본값. 사용자가 더 많은 옵션을 원하면 후속 질문으로 자연스럽게 확장

 - 강함: "**결론: 현 제품은 중단하고 글리시네이트 형태로 교체하세요.** 이유는 ① 상한 초과로 설사 부작용 가능 ② 산화 형태 흡수율 4%라 실제 흡수는 약 16mg에 불과 ③ 결과적으로 권장량 대비 실제 흡수량은 크게 부족"

### G2. 트레이드오프 있는 권고는 A/B 비교표 + 사용자 선택 질문 권장

- **적용 조건**: 예산·효능·접근성이 **뚜렷하게 충돌**하고, 사용자 선호가 결정 요인인 경우 (예: 현 제품 재구매 2만원 vs 파이토솜 업그레이드 6만원, 흡수율 7~10배)
- **적용하지 않음**: 
 - 명확한 정답이 있는 교체 (예: 산화마그네슘 → 글리시네이트, 비타민 D 중복 제거) → 단독 권고
 - 안전성 이슈 (예: UL 초과) → 단독 권고 (선택의 여지 없음)
 - 비용·접근성 차이가 미미한 경우 → 단독 권고
- **비교표 컬럼**: 선택지명 / 제품 예시 / 장점 / 단점 / 월 비용 대략
- **마무리 문구**: "어느 쪽으로 진행할지 알려주시면 해당 옵션 상세를 제시해드립니다" 1문장
- **기본값**: 애매하면 **단독 권고**. 사용자가 더 많은 옵션을 원하면 후속 질문으로 확장

### 가이드라인 신규 등재 원칙 (L13 교훈)

스킬이 범용 배포 대상임을 고려해 새 가이드라인은 다음을 통과한 경우만 등재:
1. **범용성**: 특정 사용자 취향이 아닌, 다양한 프로필에 공통적으로 적용되는가
2. **반복성**: 2회 이상 서로 다른 테스트에서 동일 패턴이 확인됐는가 (1회성 피드백은 §9 changelog에 "관찰 사례"로만 남김)
3. **충돌 점검**: 기존 §8 절대 금지나 다른 가이드라인과 상충하지 않는가
4. **기본값 선택**: 규칙 적용 vs 미적용 중 어느 쪽이 더 많은 사용자에게 유리한가
